CSys class

this is just a collection of hints and tips. You should open util_sys.h and study interface :)

This class is singleton. if you want to access its functions, just use global sys instance

Beware that if you wish to use it inside some static function, you must use CSys::getInstance() to acquire CSys instance instead, because you cannot assume sys instace will be ready for you so soon.

In order to detect CPU speed, CSys must wait for about 200ms (400 under linux). So program will start bit later, it is normal

Hints

  • obtain CPU speed with getCpuFrequency()
  • actual CPU sysle cound can be obtained with getCycles()
  • getDate() and getTime() convert time variable to ISO format string
  • getProcessId() and getThreadId() returns current process/thread identifier

Profiler how to

  1. declare uint64 variable (e.g. cycles) and store result from getCycles()
  2. do some time consuming stuff
  3. cycles = getCycles() - cycles; and you'll have duration in CPU cycles
  4. convert from cycles to (mili)seconds with e.g. computeMiliSecondsDouble()
