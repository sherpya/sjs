CPar class

this is just a collection of hints and tips. You should open util_par.h and study interface :)

CPar is most complex utility. It supports xml, ini and cfg files. You can ask tree-like queries into xml (oh excuse my poor english :))

basic features are here : http://neuron.tuke.sk/~wagner/files/parser/index.html, but that help is HEAVILY OUTDATED :) Syntax has changed slightly some features added, some removed ...

Some basics

  • loadXML, loadCFG, loadINI, saveXMl, saveCFG, saveINI
  • isSet returns if element is present
  • getValue returns value as a string
  • getValueEx returns value as a anything
  • setValue, setValueEx, setName
  • insertElement, insertAttribute, erase
  • moveUp, moveDown, moveFront, moveBack
  • use Hold() and Reset() to speed up searching in tree. After Hold() search will begin on holded element, acquired by getElement()

Query

  • It is like directories
  • child element by /
  • go up one element with ^
  • go to attribute with :
  • test for value with =
  • you can use ! & and |
  • example : bot/name='Frodo'
  • example : server/ip!='192.168.1.1'

I do not recommend mixing file types (e.g. loading xml and saving it as cfg)

See par and test example and learn from them
