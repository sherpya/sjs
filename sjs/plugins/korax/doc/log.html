CLog class

this is just a collection of hints and tips. You should open util_log.h and study interface :)

CLog is singleton class. if you log in static function, don't use LOG_* macros, use CLog::getInstance()::Report() instead

set directory in util_log.cpp by changing directory_ constant variable in CLog constructor

CLog logs following things

  • custom text
  • time, date, process id, thread id
  • source file name, line in source file, name of function

macros

  • LOG_INFO - normal, log to log.xml
  • LOG_WARNING - warning, log to log.xml and cerr.log
  • LOG_ERROR - error, log to log.xml and cerr.log
  • LOG_FUNCTION - log function enter/exit, log to log.xml

anything you send to std::cout will be redirected to cout.log

set buffer sizes in CLog constructor. 0 is recommended for cerr, 4096 for other 2. If your program crashes while debugging, log.xml will be corrupted and you must set buffer to 0 if you want to know last logged information. However 0 buffer results in slow log, so use buffer (e.g. 4096 bytes big) in retail release.

CLog has built-in support for koraX's build number logger, enable it in util_log.cpp

Customize what will be saved to log.xml by modifying Report() function in util_log.cpp
